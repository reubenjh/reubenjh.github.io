JavaScript Language

Explaining all the use cases of:

Parentheses: ()

  • Parentheses are used to surround a function's arguments - the information that is passed to the function to work with.

    e.g. function myFunction (spoon, fork)

    When you call myFunction, you pass it the info representing the fork and the spoon e.g. myFunction ('airNZ branded fork', 'silver spoon') to perform its task on them. This is true for predefined functions too such as push() or addEventListener().

  • They're also used to define conditions or parameters in if, for or while loops. e.g. if (x === 10) {piece of code} will only execute the code if the condition that x strictly equals 10 in the parenthesis is true.

  • They can also be used to control the order in which calculations are done in more complicated maths problems. If a piece of arithmetic is in parenthesis, it will be calculated first. If there are parenthesis within those parenthesis, then those will be calculated first out of everything in the initial parenthesis.

    e.g. in let myAnswer = x * ((x + 2) / 2), JavaScript will see the x + 2 statement is in parenthesis nested in parenthesis and calculate that, before dividing by two and finally multiplying the product by 10 to give myAnswer.
Brackets: []

  • Brackets are used to put data into an array e.g. let myArray = ['dogs', 'cats', 'fish'].

  • Brackets are also used to access data items in an array using the index, or numerical position in the array. e.g. myArray[1] is 'cats' as JavaScript counts the index number starting from 0 and 'cats' is the second data item from the left.

  • Another use for brackets similar to the above is to access properties of an object by using the property name in brackets e.g. myObject['legs'] targets the property called 'legs' in the object myObject.
Braces: {}

  • Braces are used to put data into an object e.g. let myObject = {name: 'Rufus', species: 'dog', legs: 4, bestfriend: 'man'}.

  • Braces can also be used to wrap code that will only be executed under certain circumstances. e.g. if (something) {code to execute} will only execute the code in the braces if the conditions in the parenthesis evaluate to true.

  • They're also used to define what a function does. Once the function has been defined e.g. function myFunction (one, two) {let three = one * two; return three}, it can be called at any time with its name myFunction and its arguments in parenthesis to execute the code that has been defined for it in the braces.
Single quotes: ''

  • Single quotes are used most commonly, and interchangably with double quotes "", to wrap data that is a string. In the object myObject above that is defining a dog named Rufus, Rufus needs to be in quotes as it's a word, not a number or a true/false value or anything like that.

  • In the same way as above, they're used to define words for functions to look for e.g. document.getElementById('title') will look for an element in the document with id='title'.

  • They're also used to wrap JavaScript events such as 'click', 'dblclick' or 'mouseenter'. e.g. document.getElementById('title').addEventListener('click', myFunction()) will make the element with the title id perform myFunction() when it is clicked.
Double quotes: ""

Exactly the same as above I believe, except it can be handly to have two ways to do this. If a string has quotes in it e.g. let myString = 'Geoff likes to think he is 'the Fonz'', the ' before 'the fonz' will actually close the string and it will just read 'Geoff likes to think he is ' then 'the fonz' will cause an error. Instead, we can write let myString = 'Geoff likes to think he is "the Fonz"' for JavaScript to realise the quotes are a part of the string. It can be done around the other way too, with double quotes wrapping the strong and single quotes in the sentence. As long as they are different.
