What is 'scope', and how does it work in JavaScript?

Scope is basically how visible something is. In JavaScript, there are two levels of scope as far as I know. Local scope and global scope.

As the name suggests, global scope basically means it's visible everywhere. So you can interact with it directly anywhere in your code even when you're deep in layers of other stuff. Local scope on the other hand means it's only visible within it's native block of code. The most common case where variables only have local scope is within something called 'functions'. These are little bite sized pieces of code you can write to be executed with just a keyword. New variables you define within your function have local scope so if you try to interact with them outside of your little function block, JavaScript won't be able to see them.

This all sounds like it won't be that important but it definitely is. When trying to make the history bar in my calculator work correctly, I kept running into trouble within the function that would loop through my array of entries the user has input to write each item to a string variable called history, then write history to the innerHTML of my history bar on the page. The problem was if I had it set to write each entry to the variable history, then entry 2 would overwrite entry 1, and entry 3 overwrite entry 2 etc and I would end up with just the last entry that it set, rather than the actual set of entries that have been input. If I had it append each entry onto history as it looped, I would end up with the right thing the first time I tried to write history, but next time I wanted a fresh history it woud append it again as it would look and see history is no longer empty and append the new entries onto the end of the previous ones. The issue was that history had global scope, as it was defined outside of the function so that other things could interact with it, but what I really needed was a fresh variable each time the function was called. So I made a variable within the function called tempHistory, initialized it to be empty and appended each entry to it before writing tempHistory to history, and finally writing history to the innerHTML of my history bar to display it. Since it would operate on a fresh tempHistory variable each time, it worked. I can't remember why I didn't just reset history to be empty as a part of the code before looping through and appending, which sounds like it would have been easier now that I write this, but there was definitely a reason at the time!
